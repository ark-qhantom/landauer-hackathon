## When to Use
- Client wants premium, consistent visual assets (hero marks, tiles, key art, details) for a film title sequence, game UI/props, art print series, brand kit, etc.
- Budget and timeline are tight for traditional specialists.
- The work must feel like a coherent fragment of a larger luminous, hopeful, serious, inhabitable mythic/sacred world — not generic AI output.
- The commission is micro (one hero + 5-7 supporting pieces + brief) so it can be fulfilled reliably and compounded.

Trigger phrases: "fulfill a dreamy mythic asset commission", "generate assets in the clean dreamy Catholic Greek style", "create a high-craft micro pack for an indie creator using the bible".

## The Style Bible (MANDATORY — load and obey exactly every time)
See the full `style-bible.md` in the Protos hackathon package. Key rules (never violate):

- Clean dreamy: luminous pearl/ivory/silver fields, warm gold/brass tracery, soft cathedral arches blended with Greek temple columns and friezes, ethereal but grounded heroic or abstract symbols (muses/daimons as artisan guides), soft divine/twilight glows (respectful, no parody Christian iconography or messianic figures), structured compositions (frieze bands, rose-window symmetries, key patterns).
- Anti-slop contract:
  - Every element intentional and symbolic (tells a tiny story of creation, threshold, journey, or triumph).
  - High craft: clean edges, intentional lighting (soft golden-hour + gentle rim/auroral glow), harmonious proportions, no filler or noise.
  - Consistency: same palette, light direction, motif density, symbolic language across a pack.
  - Client fit: serves the brief without over-embellishing or generic tropes.
  - Multiple passes + self-critique against the bible before final.
- Palette reference: pearl #f5f0e6 / ivory, gold #c9a24b, deep indigo #0a1020, soft lavender/blue glows #9fcfff.
- Output: High-res PNG (hero + tiles), optional ASCII/halftone variant for "sacred machine" feel, structured brief with symbolism + usage notes.

## Step-by-Step Workflow (follow exactly)
1. **Load context**: Read the style bible + any provided client brief + previous successful commissions in the same style (for compounding).
2. **Research & reference**: Use web_search or browser if needed for client context (film/game genre, existing assets). Load 1-2 consistent reference images or descriptions (e.g. from pixel-qhantom or prior packs). Note motifs that will be used.
3. **Plan the pack**: Define the hero concept (central symbol that carries the mythic/sacred story), 5-7 supporting tiles (variations that maintain perfect consistency), and the brief structure (The Commission, Symbolism & Motifs, Craft Notes, How to Use in Your Project).
4. **Generate hero**:
   - Craft extremely specific prompt that bakes in the bible rules, references, client fit, and "high-craft non-slop" (e.g. "luminous pearl field, warm gold tracery like illuminated manuscript and Greek key border, soft cathedral arch framing a heroic daimon artisan figure in temple-cathedral hybrid, soft divine twilight glow, symbolic depth showing threshold of creation, clean dreamy, masterful composition, no generic AI artifacts, consistent with previous pack").
   - Use Hermes image generation tool (or equivalent high-quality model). Request high resolution.
   - Self-critique: Does it pass every anti-slop rule? Adjust prompt and regenerate if needed (show the iteration).
5. **Generate supporting tiles**: Same prompt base + specific variations (close-up detail, different angle, social crop, in-game prop version). Enforce identical lighting, palette, motif language.
6. **Create the brief**: Structured, insightful (connects Catholic sacred geometry and Greek heroic narrative freshly), actionable for the client. Include exact usage guidance.
7. **Quality review (critical gate)**: Before delivery, present the full pack + brief to the human with a short report against the bible ("symbolic depth: yes, the hero tells a story of [X]; consistency: perfect; craft: clean edges and intentional light; no slop detected"). Only proceed after explicit human approval on quality and fit.
8. **Package & deliver**: Output the files (named consistently), the brief as markdown, and a short "value delivered" note (how this helps the client pursue their big idea).
9. **Compounding (after human sign-off)**: Analyze what worked (specific prompt refinements, motif combinations that gave high value, client reactions if any). Generate or append to a new version of this skill (or a client-specific variant) with the learnings. The next micro-commission will be faster and higher quality because of this.
